# Source Notes: essential-complexity

- Primary source: No Silver Bullet: Essence and Accidents of Software Engineering
- Author: Frederick P. Brooks Jr.
- Year: 1986
- URL: https://www.cs.unc.edu/techreports/86-020.pdf

## Key Concepts

- Essential complexity belongs to the problem itself.
- Accidental complexity comes from tools, representations, ceremony, duplication, or stale abstractions.
- Simplification is harmful when it erases real domain distinctions.

## Agent Translation

Use it to classify complexity, remove accidental parts, and keep essential rules visible.

## Common Misuse

Do not flatten domain rules just to make code look shorter.

## Source Mapping

Essential rule kept -> accidental machinery removed -> behavior proof.
