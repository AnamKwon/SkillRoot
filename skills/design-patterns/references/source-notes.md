## Key Concepts

- A pattern is a named solution to a recurring problem in context.
- The forces and consequences matter more than the pattern name.
- Patterns improve communication only when they fit the present collaboration structure.

## Agent Translation

Use it to identify forces first, compare local idioms, then choose a pattern or a simpler alternative.

## Common Misuse

Do not add abstract participants for one caller or choose a pattern before naming the problem.

## Source Mapping

Recurring problem -> forces -> pattern or simpler alternative -> consequence check.
